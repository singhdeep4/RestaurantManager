# Project Report: Restaurant Manager

## Overview
This project is a full-stack CRUD application for managing a restaurant, built with Node.js, Express, and MySQL. It provides APIs and a web interface for handling customers, staff, menu items, and orders.

---

## Database Design
The core of the project is the MySQL database, defined in `database/RestaurantManager.sql`. The schema includes the following:

### Tables
- **CUSTOMER**: Stores customer details (ID, name, phone, email, created date).
- **STAFF**: Stores staff details (ID, name, role, phone, salary, hire date, active status).
- **FOOD_ITEMS**: Menu items with name, price, category (Veg/Non-Veg/Beverage/Dessert), and availability.
- **ORDERS**: Each order links to a customer and staff, with date, total amount, and status (Pending, Preparing, Served, Paid, Cancelled).
- **ORDER_DETAILS**: Line items for each order, linking orders to food items, with quantity and unit price.

### Relationships
- Orders reference both customers and staff.
- Order details reference both orders and food items.

### Views
- **vw_order_summary**: Combines order, customer, and staff info for reporting.
- **vw_order_detail_full**: Shows detailed order lines with food info and totals.

### Triggers
- **trg_after_insert_detail**: Updates order total after adding an order detail.
- **trg_after_delete_detail**: Updates order total after removing an order detail.

### Stored Procedure
- **sp_place_order**: Places a new order with multiple items in a single transaction using JSON input.

### Seed Data
- Example customers, staff, food items, and sample orders are pre-inserted for testing/demo.

## Application Structure
- **db.js**: Sets up a MySQL connection pool using environment variables.
- **server.js**: Main Express server, sets up middleware, static file serving, and API routes.
- **routes/**: Contains route handlers for customers, staff, menu, orders, and dashboard.
- **public/**: Frontend files (HTML, CSS, JS).

---

## How the Database is Used
- All CRUD operations for customers, staff, menu, and orders are performed via REST APIs.
- The backend uses the connection pool in `db.js` to interact with the MySQL database.
- Triggers and stored procedures ensure data integrity and simplify order processing.

## Getting Started
1. Import `database/RestaurantManager.sql` into your MySQL server.
2. Configure `.env` with your DB credentials (see `db.js`).
3. Run `npm install` to install dependencies.
4. Start the server with `npm start`.
